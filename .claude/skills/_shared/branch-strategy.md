# ブランチ戦略（共通リファレンス）

## ブランチ構成

- `main`: 本番環境（必須）
- `staging`: ステージング環境（プロジェクトによっては存在しない場合がある）
- `feature/*`: 新機能開発
- `bugfix/*`: バグ修正
- `hotfix/*`: 緊急修正
- `refactor/*`: リファクタリング
- `chore/*`: 設定変更・ビルド関連
- `test/*`: テストコード
- `docs/*`: ドキュメント作成・更新
- `tmp/*`: 一時作業

## ベースブランチ選択ルール

1. `git branch -a` で利用可能なブランチを確認
2. 以下のルールでベースブランチを決定:
   - `hotfix/*` → 常に `main` から分岐し `main` へマージ
   - **その他すべて** → `staging` が存在する場合は `staging`、存在しない場合は `main`

## 永続ブランチの保護

以下のブランチは削除・force push しない:
- `main`, `staging`, `develop`
