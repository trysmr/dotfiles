# コミット 使用例

タイトルは「why中心」、本文は1段落・1行で書き切る、箇条書きはwhatでよい。

## パターンA: タイトルのみ（小さな変更）

タイポ修正、設定の微調整、依存ライブラリの軽微なバージョンアップなど。

```bash
git commit -m "READMEのインストール手順のタイポを修正"
```

```bash
git commit -m "Rubocopのターゲットバージョンを3.4に更新"
```

## パターンB: タイトル + 本文

背景や経緯を補足したい場合。本文は1段落・1行で書き切る。

```bash
git commit -m "$(cat <<'EOF'
検索UIの応答速度を改善するためページネーションを実装

ヒット件数が多い場合に全件表示で初期描画が数秒かかる問題があったため、20件ずつの段階的取得に変更。
EOF
)"
```

## パターンC: タイトル + 本文 + 箇条書き

複数の関連する変更を1コミットでまとめる場合。

```bash
git commit -m "$(cat <<'EOF'
ユーザー検索体験を改善

大量ユーザー一覧から目的の人を素早く見つけられるよう、検索とページネーションを追加。

- 名前・メールアドレスでの検索に対応
- ページネーション実装
- 検索条件の保持機能追加
EOF
)"
```

## 省略パターン: タイトル + 箇条書きのみ（本文省略）

タイトルだけで背景が伝わる場合は本文を省略可。

```bash
git commit -m "$(cat <<'EOF'
依存ライブラリを最新の安定版に更新

- nokogiriを1.18.0に更新
- jbuilderを2.13.0に更新
- pumaを6.5.0に更新
EOF
)"
```

## アンチパターン

### what中心のタイトル

```
NG: ユーザー検索機能を追加
OK: 大量ユーザー一覧から目的の人を素早く見つけられるよう検索機能を追加
```

### 本文を複数行に分割

```
NG:
  ヒット件数が多い場合に
  全件表示で初期描画が遅い問題があったため、
  20件ずつの段階的取得に変更。

OK:
  ヒット件数が多い場合に全件表示で初期描画が遅い問題があったため、20件ずつの段階的取得に変更。
```
