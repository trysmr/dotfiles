# Commit Workflow

このスキルは現在の変更をGitブランチにコミットします。

## 実行手順

### 1. ブランチ戦略の確認

ブランチ構成とベースブランチ選択ルールは `.claude/skills/_shared/branch-strategy.md` を参照。

```bash
git branch -a
```

で利用可能なブランチを確認します。

### 2. 保護ブランチチェック

現在のブランチを確認します：

```bash
git branch --show-current
```

ステップ1で確認したブランチ戦略に基づいて判断します：

- **ブランチ戦略が存在し、現在のブランチが作業ブランチの場合**: そのまま次の手順に進みます。
- **ブランチ戦略が存在し、現在のブランチが保護ブランチの場合**: `create-branch`スキルを実行してください。ブランチ作成が完了してから次の手順に進みます。
- **ブランチ戦略が存在しない場合**: ブランチを作成するか、このまま直接コミットするか、ユーザーに確認してください。

### 3. 変更内容の確認

```bash
git status
git diff
```

で変更ファイルと差分を確認します。

### 4. コミット情報の準備

変更内容に基づいて、以下を準備します：

```bash
# ステージング
git add <files>

# ステージング内容の確認
git diff --cached
```

コミットメッセージ案を作成：
- 基本は50文字程度の**1行**で記述
- ただし、変更が多い場合は変更内容の具体的な説明を箇条書きしてもよい
- 「何を変えたか」ではなく「**なぜ変えたか**」を重視して記述
- 英語と日本語の間に余計な半角スペースを追加しない

### 5. ユーザー確認

以下の git commit コマンド形式でユーザーに提示し、コミットの許可を得てください：

```bash
git commit -m "なぜ変更したかを1行で記述"
```

#### 変更が多い場合

```bash
git commit -m "$(cat <<'EOF'
なぜ変更したかを1行で記述

- 変更内容の詳細1
- 変更内容の詳細2
EOF
)"
```

**変更ファイル**:
- ファイル1
- ファイル2
- ...

**重要**: ユーザーの許可なしにコミットを実行しないでください。

### 6. 計画ファイルのTODO更新確認

`~/.claude/plans/`に計画ファイルが存在する場合、コミットする変更に対応するTODO項目を確認します：

```bash
ls ~/.claude/plans/*.md 2>/dev/null
```

計画ファイルが存在する場合：
- 今回のコミットで完了したタスクに対応する`- [ ]`を`- [x]`に更新する
- 設計変更で不採用になった項目は`~~取り消し線~~`で明示する
- ユーザー確認が必要で完了判定できない項目は`- [ ]`のまま残す

### 7. コミット実行

ユーザーの許可を得てから、コミットを実行します：

```bash
git commit -m "なぜ変更したかを1行で記述"
```

#### 変更が多い場合

```bash
git commit -m "$(cat <<'EOF'
なぜ変更したかを1行で記述

- 変更内容の詳細1
- 変更内容の詳細2
EOF
)"
```

コミットメッセージは日本語で記述します。

## チェックリスト

実行前に確認すべき項目：

- [ ] ブランチ戦略（`branch-strategy.md`）を理解した
- [ ] `git branch -a`で利用可能なブランチを確認した
- [ ] `git status`と`git diff`で変更内容を確認した
- [ ] 保護ブランチチェックを実施した（ブランチ作成済み、またはユーザーが直接コミットを指示）
- [ ] セキュリティ情報（`.env`, `credentials`等）を含まない
- [ ] ステージング済みファイルとコミットメッセージ案をユーザーに提示した
- [ ] ユーザーの許可を得てからコミットを実行した
- [ ] コミットメッセージが「なぜ変更したか」を日本語で明確に記述されている
- [ ] 計画ファイルが存在する場合、完了したTODO項目を`- [x]`に更新した

## Troubleshooting

| エラー | 原因 | 対処 |
|--------|------|------|
| pre-commitフックがコミットを拒否する | lintエラー・secretが含まれている等 | エラーメッセージを確認して修正後に再コミット（`--no-verify`は使わない） |
