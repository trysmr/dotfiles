# record-lesson: 教訓記録スキル

## 目的

Mitchell Hashimotoの原則「エージェントがミスしたら二度と繰り返さない仕組みを作る」を実現する。
autoMemoryによる日常的な学習とは異なり、このSkillは**重要な教訓を構造化して永続的に記録**する。

## 実行手順

### ステップ1: 教訓の分類

ユーザーとの会話から、以下のどのカテゴリに該当するか判断する:

- **feedback**: アプローチの修正・確認（作業の進め方、コードスタイル、ツールの使い方）
- **project**: プロジェクト固有の事実・決定（設計判断、制約、背景情報）
- **user**: ユーザーの役割・知識・好みに関する情報
- **reference**: 外部リソースへのポインター（ドキュメント、ツール、ダッシュボード）

### ステップ2: 教訓の構造化

ユーザーに以下を確認・整理する:

1. **何が起きたか**: 具体的な事象（エラー、非効率、発見）
2. **なぜ起きたか**: 根本原因（前提の誤り、知識不足、パターンの見落とし）
3. **どう防ぐか / 今後どう活かすか**: 具体的な対策やルール

### ステップ3: memoryファイルの作成

以下のフォーマットでmemoryファイルを作成する:

```markdown
---
name: {{教訓の簡潔なタイトル}}
description: {{1行の説明 — 将来の会話で関連性を判断するために使う}}
type: {{feedback | project | user | reference}}
---

{{教訓の本文}}

**Why:** {{なぜこの教訓が重要か、背景の状況}}
**How to apply:** {{この教訓をいつ・どこで適用すべきか}}
```

**保存先の特定方法**:

Claude Codeのauto memoryシステムは、プロジェクトのパスをディレクトリ名に変換して保存先を決定する。
現在のプロジェクトのmemoryディレクトリを特定するには:

```bash
# PWDのパスから / を - に変換してプロジェクトハッシュを生成
PROJECT_DIR=$(echo "$PWD" | sed 's|/|-|g')
MEMORY_DIR="$HOME/.claude/projects/${PROJECT_DIR}/memory"
```

**保存先**: `~/.claude/projects/{プロジェクトパス変換}/memory/` 配下
**ファイル名**: `{type}_{snake_case_topic}.md`（例: `feedback_no_mock_database.md`）

### ステップ4: MEMORY.mdインデックスの更新

同じmemoryディレクトリの `MEMORY.md` に1行のポインターを追加する。

形式: `- [タイトル](ファイル名.md) — 1行フック（150文字以内）`

### ステップ5: ユーザーへの報告

記録した教訓をユーザーに提示し、内容が正確か確認する:

```
## 記録した教訓
- **カテゴリ**: feedback
- **タイトル**: データベースをモックしない
- **ファイル**: feedback_no_mock_database.md
- **要約**: 統合テストでは実DBを使用する。モックとの乖離で本番障害が発生した前例あり。
```

## チェックリスト

- [ ] 既存のmemoryファイルと重複がないか確認した
- [ ] 教訓の内容がコードや git history から導出できない非自明な情報である
- [ ] description が将来の関連性判断に十分な具体性を持っている
- [ ] MEMORY.mdのインデックスを更新した
- [ ] ユーザーに記録内容を提示し確認を得た
